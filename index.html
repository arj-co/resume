01

Serene / Inner Bloom

Mindful Meditation Web Application

Stack TypeScript, Vite
Focus Calm UX, Cognitive Ease

I wanted to build something that feels like a deep breath. Not another productivity app disguised as wellness, but an actual space where your mind can settle without being told what to do.

The interface strips away everything that doesn't serve stillness. No aggressive timers, no achievement badges, no guilt-inducing streaks. Just generous whitespace, gentle transitions, and typography that doesn't shout. Every interaction is designed to feel unhurried, like the app itself is breathing with you.

View the website!

Design Principles

  • Visual rhythm that mirrors natural breathing patterns
  • Typography deliberately oversized for easy, unfocused reading
  • Color palette pulled from early morning light and late evening calm
  • Motion that suggests settling, not hurrying
02

Cockpit Console

Web3 Interface Demo

Stack TypeScript, Vite
Focus Information Architecture

This started as a personal challenge: could I build a Web3 interface that doesn’t feel like it’s trying to sell you something? A lot of crypto products lean heavily on flashy gradients and animated charts, and that always felt disconnected from the fact that these tools deal with real value and real decisions.

I ended up thinking about it like a driver’s cockpit. Everything has a place you can rely on. You shouldn’t have to hunt for information or guess what a control does. The layout is kept tight and intentional, typography helps you scan quickly, and color is used sparingly—only to show when something actually changes.

View the website!

Architecture

  • Component library with consistent visual grammar
  • Real-time state management without visual chaos
  • Responsive grid that maintains information density
  • Accessibility-first interaction patterns
03

BlockElect

Blockchain-Based E-Voting System

Stack Solidity, React, Hardhat, Ethers.js
Focus Decentralization, Cryptographic Identity

Voting systems require trust, and centralized systems require trusting the people who run them. I wanted to explore whether smart contracts could enforce voting rules programmatically—not as a replacement for democracy, but as a technical exercise in trustless verification.

The entire voting logic lives on-chain. Candidate registration, voter eligibility, vote casting, and result tallying all happen in Solidity. The frontend is deliberately thin—just a way to connect your wallet and interact with the contract. MetaMask handles identity, Hardhat simulates the blockchain locally, and Ethers.js bridges the two.

What I learned: transparency doesn't come from flashy UIs. It comes from readable code and immutable state. The blockchain itself is the audit trail.

View on GitHub

Core Architecture

  • Solidity smart contract enforcing all election rules
  • Hardhat development environment with local blockchain
  • MetaMask integration for cryptographic identity
  • React interface with Ethers.js for contract interaction
  • Node.js for build orchestration and tooling

Voting Flow

  1. Admin deploys contract with candidate list
  2. Eligible voters register on-chain with wallet addresses
  3. Voter connects wallet via MetaMask
  4. Vote submission triggers smart contract transaction
  5. Contract validates eligibility and records vote immutably
  6. Results computed directly from blockchain state
