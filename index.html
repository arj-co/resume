BlockElect

Blockchain-Based E-Voting System

Stack Solidity, React, Hardhat, Ethers.js
Focus Decentralization, Cryptographic Identity

A decentralized e-voting prototype enforcing all voting rules on-chain, eliminating centralized control. The system implements trustless verification where the blockchain itself becomes the source of truth.

Smart contracts define candidate lists, voter eligibility, and immutable vote storage. The frontend serves as a thin interface layer, with MetaMask handling cryptographic identity and transaction signing.

View on GitHub →

Core Architecture

  • Solidity smart contract defining election rules
  • Hardhat local Ethereum blockchain for deployment
  • MetaMask as cryptographic identity layer
  • React + Ethers.js for wallet connection
  • Node.js runtime for orchestration

Voting Flow

  1. Contract deployment by admin
  2. On-chain voter registration
  3. Wallet connection via MetaMask
  4. Vote casting through frontend interface
  5. Smart contract validation and storage
  6. Results read directly from blockchain state
